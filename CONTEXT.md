# Context: AI-QA-Automation

A web portal that turns a product requirements document into QA artifacts through a
staged AI pipeline (orchestrated in n8n).

## Glossary

### PRD (Product Requirements Document)
The source document a user uploads (`.csv`, `.pdf`, or `.docx`). The origin of the
entire pipeline. Every downstream stage may reference it, not just the immediately
preceding stage.

### Pipeline
The ordered sequence of generation stages:

`PRD → Epics → User Stories → Test Cases`

Each stage is generated by an n8n workflow, reviewed/edited by the user, then used as
input to the next stage. Stages after the first reference **both** the previous stage's
(edited) output **and** the original PRD.

### Epic
A coarse-grained unit of work generated from the PRD. The current results/review screen
displays and lets the user edit Epics. Count is influenced by the `numberOfEpics` input
given at upload time.

### User Story
The next stage after Epics. Generated by referencing the edited Epics **and** the
original PRD. An Epic is the parent of many User Stories (one-to-many).

### Test Case
A planned future stage after User Stories. Not yet built.
